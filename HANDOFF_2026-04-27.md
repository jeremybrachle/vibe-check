## STEP 2 — Run frontend and backend separately (still one repo)

This proves the frontend has zero hard coupling to FastAPI. We'll serve the
static files via Python's built-in HTTP server on a different port and have
the browser talk to the API on `:8000` via CORS.

### 2a. In `web/runtime-config.js`, point the frontend at the API

Currently:
```js
window.VIBE_CONFIG = Object.assign(
  { API_BASE: "" },
  window.VIBE_CONFIG || {}
);
```

Change `API_BASE: ""` to `API_BASE: "http://localhost:8000"`.

Why: when the page is served from `:5173`, an empty `API_BASE` would make
fetch calls go to `http://localhost:5173/api/v1/...` which doesn't exist.

### 2b. In `.env`, lock CORS to the frontend's origin

Open `.env` and change:
```
ALLOWED_ORIGINS=*
```
to:
```
ALLOWED_ORIGINS=http://localhost:5173,http://127.0.0.1:5173
```

(`*` works too for dev, but a real list is better hygiene and matches what
prod will look like.)

### 2c. Optional but recommended — disable the FastAPI static mount

This is the single line in `app/main.py` that makes the backend "also serve
the frontend":
```python
app.mount("/", StaticFiles(directory="web", html=True), name="web")
```

Comment it out (or delete it) for this step so we know for sure the page is
being served by the standalone server, not by FastAPI.

You'll also want to drop the now-unused import:
```python
from fastapi.staticfiles import StaticFiles
```

### 2d. Start the BACKEND on port 8000 (terminal A, WSL)

```bash
cd /home/kerry/programming/vibe-check
.venv/bin/python -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

### 2e. Start the FRONTEND on port 5173 (terminal B, WSL)

```bash
cd /home/kerry/programming/vibe-check/web
python3 -m http.server 5173 --bind 0.0.0.0
```

That's it. No npm. No Vite. Three static files, served as-is.

### 2f. Verify

```bash
# In a third WSL terminal
curl -sSI http://127.0.0.1:5173/        # → HTTP 200 (the HTML file)
curl -sSI http://127.0.0.1:8000/api/v1/health   # → HTTP 200 (the API)
curl -sSI http://127.0.0.1:8000/        # → HTTP 404 (static mount is OFF)
```

In the Windows browser open `http://localhost:5173/` (incognito).

Open DevTools → Network. You should see the page request go to `:5173`
and every `/api/v1/...` request go to `:8000`. If you see CORS errors in the
console:
- The `ALLOWED_ORIGINS` change in 2b didn't take effect — restart uvicorn
  (Ctrl+C and re-run 2d). uvicorn `--reload` reloads on `app/` changes but
  not always on `.env` changes.

## Risks and how we mitigate them (so future-Kerry doesn't yell at past-Kerry)

| Risk | Mitigation |
| --- | --- |
| Frontend ships expecting an API field the backend renamed | When you change a `/api/v1/...` response shape, grep the web repo for the field name. We can add a contract test later (one Python script that hits the live API and validates JSON keys exist). |
| Forgot to update `API_BASE` after deploying the API somewhere new | The frontend has a single source of truth for the host (`runtime-config.js`). Document it in the web repo's README. |
| CORS misery | `ALLOWED_ORIGINS` in API `.env` must list every origin (localhost dev + deployed frontend URL). Browser DevTools → Network → red rows tell you exactly which origin is being rejected. |
| Two repos = two deploys = forgetting one | Keep them dumb-simple. API is one Docker container. Frontend is just static files. No CI required for v1; deploy by hand if needed. |
